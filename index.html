/Overview/HTML5 History API

Front-end developer Christian Shum-Harden of RED Interactive Agency demonstrates some best practices and strategies when preparing to use the HTML5 History API.

Knowledge Needed HTML, JavaScript
Requires Text editor, modern web browser, local webserver
Project Time 1 hour

The HTML5 History API is a small collection of methods and events that enable us to manipulate the browser location and history without triggering a page refresh.

The History API is really small; at its core pushState changes the browser location-bar and triggers a popstate event. 

window.addEventListener("popstate", function(e) { // list for the change
	// location has changed
});

history.pushState({ foo: "bar"}, "Example Title", "/change/the/location"); // request a location change
The URL in our location-bar changed and the browser doesn't reload. For such a simple result, the outcome is strikingly important; we've just inherited the most crucial responsibility of the internet - we must respond to a URL request with an accurate result.

URLs are the fabric of the internet so we'll want to think critically about our approach and goals for overriding years of browser experience and care. This tutorial aims to provide a foundation of knowledge and thoughtfulness around the History API by:

  • Studying URL design and the affectiveness of a traditional web architecure with and without the History API
  • Using the History API to develop a flexible but reliable site framework that promotes the iterative improvement of load-time, enhanced interactivity, and elegant page transitions.
  • Approaching pushState from an adaptive perspective that balances the tradeoffs of missing or varying browser History API implementations.

If you'd prefer to slam your way through pushState, then have a look at the ajaxify-html5.js script that "ajaxifies your entire website instantly". It "hijacks" all your site URLs and cross-fades between page requests.
